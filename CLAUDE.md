# mfyz-resume

## Project Structure

Monorepo with two packages sharing a single `resume.yaml` at the root.

- `packages/cli/` — Published as `fatih-yildiz` on npm. Zero-dependency CLI resume (`npx fatih-yildiz`). Vanilla JS only — `fs` and `path`, no frameworks.
- `packages/generator/` — Private. Uses yamlresume for PDF/HTML/Markdown generation. Templates: calm (HTML), moderncv-classic (PDF).
- `resume.yaml` — Single source of truth. Follows yamlresume schema with custom fields (e.g., `location`) that yamlresume ignores but CLI uses.

## Publishing

- Package name: `fatih-yildiz` (npm blocked `mfyz` and `ffyz` due to similarity rules)
- `prepublishOnly` copies root `resume.yaml` into `packages/cli/` before publish
- Bump version in `packages/cli/package.json`, then `cd packages/cli && npm publish`
- Both `packages/cli/resume.yaml` and root `resume.yaml` must stay in sync

## CLI Design

- 80-char width, 1-char outer padding
- ANSI 256-color: sky blue (#33) headers, orange (#208) numbers, pink/magenta company names
- Year-only dates in CLI output (source YAML keeps full month precision)
- Number highlighting regex handles `$`, `+`, `%`, `x`, `bn`, `k`, `m` suffixes
- No footer, no version display

## Generator

- Post-build `strip-branding.js` removes "Generated by yamlresume" lines from HTML/MD output
- PDF requires tectonic (`brew install tectonic`)
